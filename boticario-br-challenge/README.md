## 🎯 Project Objectives
The goal of this project was to transition from raw, disparate data formats into a highly consolidated analytical data lake, and then hook it up to external social platforms for sentiment tracking. 

We accomplished this via a two-layer architecture:
1. **Sales Consolidation:** Ingesting 3 years (2017-2019) of disparate sales CSV files into a centralized PySpark engine. Cleaning and aggregating them into 4 distinct executive views (Dates, Brands, Lines, and overall Sales Volume).
2. **Dynamic Social Ingestion:** Dynamically querying the gold tables to find the highest-grossing product of December 2019, and injecting that string into a Twitter API hook to ingest real-world external sentiment straight into our Data Lake.

## 🛠️ The Technology Stack
* **Cloud Platform:** Databricks (Serverless / Clusters)
* **Data Ingestion Engine:** PySpark
* **External Integration:** Tweepy (Twitter API v2)
* **In-Memory Manipulation:** Pandas
* **Presentation:** HTML, CSS Glassmorphism, UI/UX Animations

## 📁 Repository Structure
- `/src/etl_sales.py`: The PySpark backbone. Connects to the raw layer, cleans dates, aggregates sales metrics, and filters out our "winning" product.
- `/src/twitter_scraper.py`: A robust script interacting with Twitter. Automatically throttles requests and extracts real-time product mentions.
- `/data/`: Contains the Bronze (raw CSVs) and Silver/Gold artifacts generated by the PySpark logic.
